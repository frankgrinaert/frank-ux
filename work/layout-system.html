Case study

Revised web layout system

Replaced an over-engineered column grid in XMC Pages with a simpler layout model — easier authoring, faster responsive builds, fewer dead ends for marketers.

Sitecore XMC Pages Feature design

1 · Business problem

Complexity was blocking content teams

XMC Pages is Sitecore’s site builder. The legacy layout system used a configurable column grid that was simultaneously hard to learn and too rigid — e.g. a five-column section on a twelve-column grid was awkward or impossible. Responsive behaviour was tedious to author.

That friction slows campaigns, increases training cost, and pushes power users toward workarounds. Product needed a model that does less but does it better.

2 · Product gap

Power without usability

Authors needed flexible page structure without becoming layout engineers. Engineering needed implementable rules that would not explode into edge cases per customer site.

3 · Approach

New layout model + clear specs

I designed a simplified layout system and presented it to the squad with implementation-ready specs.

Layout system prototype (Figma) →

4 · Outcomes

Expected impact

Reduced authoring time for responsive pages, fewer support escalations on “why can’t I build this layout?”, and a cleaner contract for engineering to maintain. Success is measured in shipping velocity and author satisfaction, not pixel awards.
